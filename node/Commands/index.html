    Commands

    Enumerations

    Enumeration Description
    BitmapIndexType Enumeration specifying if index arguments are BYTE indexes or BIT indexes. Can be specified in BitOffsetOptions, which is an optional argument to the bitcount command. Can also be specified as an optional argument to the BaseClient.bitposInverval|bitposInterval command.
    BitOverflowControl Enumeration specifying bit overflow controls for the bitfield command.
    BitwiseOperation Enumeration defining the bitwise operation to use in the bitop command. Specifies the bitwise operation to perform between the passed in keys.
    ConditionalChange An optional condition to the geoadd, zadd and set commands.
    ExpireOptions -
    FlushMode Defines flushing mode for GlideClient.flushall, GlideClusterClient.flushall, GlideClient.functionFlush, GlideClusterClient.functionFlush, GlideClient.flushdb and GlideClusterClient.flushdb commands.
    FunctionRestorePolicy Option for FUNCTION RESTORE command: GlideClient.functionRestore and GlideClusterClient.functionRestore.
    GeoUnit Enumeration representing distance units options.
    InfBoundary -
    InfoOptions INFO option: a specific section of information: When no parameter is provided, the default option is assumed.
    InsertPosition Defines where to insert new elements into a list.
    ListDirection Enumeration representing element popping or adding direction for the List Based Commands.
    ScoreFilter Mandatory option for zmpop. Defines which elements to pop from the sorted set.
    SortOrder Defines the sort order for nested results.
    TimeUnit Time unit representation which is used in optional arguments for getex and set command.
    UpdateByScore Options for updating elements of a sorted set key.

    Classes

    Class Description
    BitFieldGet Represents the "GET" subcommand for getting a value in the binary representation of the string stored in key.
    BitFieldIncrBy Represents the "INCRBY" subcommand for increasing or decreasing bits in the binary representation of the string stored in key.
    BitFieldOverflow Represents the "OVERFLOW" subcommand that determines the result of the "SET" or "INCRBY" bitfield subcommands when an underflow or overflow occurs.
    BitFieldSet Represents the "SET" subcommand for setting bits in the binary representation of the string stored in key.
    BitOffset Represents an offset in an array of bits for the bitfield or bitfieldReadOnly commands.
    BitOffsetMultiplier Represents an offset in an array of bits for the bitfield or bitfieldReadOnly commands. The bit offset index is calculated as the numerical value of the offset multiplied by the encoding value.
    SignedEncoding Represents a signed argument encoding.
    UnsignedEncoding Represents an unsigned argument encoding.

    Interfaces

    Interface Description
    BaseScanOptions This base class represents the common set of optional arguments for the SCAN family of commands. Concrete implementations of this class are tied to specific SCAN commands (SCAN, SSCAN).
    BitEncoding Represents a signed or unsigned argument encoding for the bitfield or bitfieldReadOnly commands.
    BitFieldOffset Represents an offset for an array of bits for the bitfield or bitfieldReadOnly commands.
    BitFieldSubCommands Represents subcommands for the bitfield or bitfieldReadOnly commands.
    BitOffsetOptions Represents offsets specifying a string interval to analyze in the bitcount and bitpos commands. The offsets are zero-based indexes, with 0 being the first index of the string, 1 being the next index and so on. The offsets can also be negative numbers indicating offsets starting at the end of the string, with -1 being the last index of the string, -2 being the penultimate, and so on.
    ClusterScanOptions Options for the SCAN command. match: The match filter is applied to the result of the command and will only include keys that match the pattern specified. count: COUNT is a just a hint for the command for how many elements to fetch from the server, the default is 10. type: The type of the object to scan. Types are the data types of Valkey: string, list, set, zset, hash, stream. allowNonCoveredSlots: If true, the scan will keep scanning even if slots are not covered by the cluster. By default, the scan will stop if slots are not covered by the cluster.
    CoordOrigin The search origin represented by a GeospatialData position.
    FunctionListOptions Optional arguments for FUNCTION LIST command.
    GeoAddOptions Optional arguments for the GeoAdd command.
    GeoBoxShape Rectangle search shape defined by the width and height and measurement unit.
    GeoCircleShape Circle search shape defined by the radius value and measurement unit.
    GeospatialData Represents a geographic position defined by longitude and latitude. The exact limits, as specified by EPSG:900913 / EPSG:3785 / OSGEO:41001 are the following:
    Limit The LIMIT argument is commonly used to specify a subset of results from the matching elements, similar to the LIMIT clause in SQL (e.g., SELECT LIMIT offset, count).
    LolwutOptions Additional parameters for LOLWUT command.
    LPosOptions Optional arguments to LPOS command.
    MemberOrigin The search origin represented by an existing member.
    RangeByIndex Represents a range by index (rank) in a sorted set. The start and end arguments represent zero-based indexes.
    RestoreOptions Optional arguments for RESTORE command.
    ScanOptions Options for the SCAN command. match: The match filter is applied to the result of the command and will only include keys that match the pattern specified. count: COUNT is a just a hint for the command for how many elements to fetch from the server, the default is 10. type: The type of the object to scan. Types are the data types of Valkey: string, list, set, zset, hash, stream.
    SetOptions -
    SortOptions Optional arguments to sort, sortStore and sortReadOnly commands.

    StreamClaimOptions Optional parameters for xclaim command.
    StreamGroupOptions Optional arguments for xgroupCreate.
    StreamPendingOptions Optional arguments for xpending.
    StreamReadOptions Optional arguments for xread command.
